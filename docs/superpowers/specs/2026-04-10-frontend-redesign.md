## 一、现状分析

### 1.1 当前前端问题诊断

经过完整分析 `deepanalyze/frontend/src` 全部源码，发现以下核心问题：

| 问题类别 | 具体问题 | 严重度 |
|----------|---------|--------|
| **消息发送Bug** | `MessageInput.tsx` 同时通过WebSocket和REST API双发消息 | 严重 |
| **XSS漏洞** | `dangerouslySetInnerHTML` 直接渲染未净化的marked输出 | 严重 |
| **死代码** | `PluginManager`/`SkillBrowser` 导入但未渲染；右侧面板逻辑完整但未使用；`lucide-react` 依赖未实际使用 | 中等 |
| **原生对话框** | 使用 `prompt()`/`confirm()` 创建知识库/删除插件，极度不专业 | 高 |
| **无国际化** | 所有文本硬编码中文，无i18n体系 | 中等 |
| **无响应式** | 侧边栏固定256px，无移动端适配 | 中等 |
| **无错误边界** | 任何组件崩溃导致整个页面白屏 | 中等 |
| **无持久化** | 页面刷新丢失全部状态 | 中等 |
| **无骨架屏** | 加载状态只有spinner或文本 | 低 |
| **无空状态设计** | 空列表只显示纯文本 | 低 |
| **图表固定尺寸** | 知识图谱Canvas固定800x600 | 低 |
| **功能缺失** | 缺少Wiki浏览、文档管理、L0/L1/L2展开、实体导航、报告导出等设计规范中要求的核心功能 | 严重 |

### 1.2 AIE_new 参考前端分析

AIE_new 是一个 **Vue 3 + Pinia** 的成熟前端系统，拥有 70+ Vue 组件，具备以下值得参考的设计特征：

**视觉设计体系**：
- 双主题系统：**浅色为默认**（白底，柔和蓝灰现代风，灵感来自GitHub/Linear/Tailwind）+ 深色（赛博朋克霓虹青 #00f0ff）
- 完整CSS变量体系：设计令牌(tokens.css)、主题(themes.css)、组件(design-system.css)
- 精细化间距/圆角/阴影/过渡系统
- 自定义滚动条样式
- 统一的组件视觉语言

**模块化架构**（14个功能模块）：
- `chat` - 聊天窗口（42K行，含6个composable拆分）
- `settings` - 设置面板（含15个子配置组件：模型/渠道/安全/嵌入/人格/工作区等）
- `knowledge` - 知识搜索系统
- `memory` - 记忆编辑/查看器
- `scheduler` - 定时任务管理（含Cron Builder）
- `skills` - 技能库/编辑器
- `tasks` - 任务看板
- `teams` - 子Agent团队管理
- `tools` - 工具面板（文件操作/Shell执行器/工具历史）
- `experience` - 体验面板
- `heartbeat` - 心跳监控
- `system` - 系统信息侧边栏

**UI组件库**（18个通用组件）：
Button, Input, Select, Modal, Toast, ConfirmDialog, ToggleSwitch, ThemeToggle, LanguageSelector, Icon, Spinner, Skeleton, EmptyState, LoadingState, DropZone, FileSelector, FilePreviewModal, ModelStatusLights

**基础设施**：
- vue-i18n 国际化（中/英双语）
- 全局确认对话框（替代原生confirm）
- 主题切换（浅色/深色）
- 完整的API层（agentTeams/knowledge/research/security/heartbeat等）

## 二、设计目标

### 2.1 核心原则

1. **专业级视觉**：参考AIE设计系统，建立统一的颜色、间距、圆角、阴影、动画体系
2. **功能完备**：覆盖设计规范文档中定义的全部8个页面模块
3. **交互精致**：每个按钮、输入框、卡片、对话框都有精心设计的交互反馈
4. **工程规范**：TypeScript严格类型、组件化架构、状态管理规范、错误处理完善

### 2.2 视觉风格定义

**设计理念**：默认以白色底的浅色主题为主，参考AIE设计系统中的浅色主题（灵感来源：GitHub, Linear, Tailwind——柔和、现代、专业）。深色主题作为可选切换。

#### 主题切换机制

```
默认行为:
  - 首次访问 → 浅色主题（白底）
  - 用户切换 → 保存到 localStorage('deepanalyze-theme')
  - 可选: 跟随系统偏好 prefers-color-scheme

切换按钮:
  - 位于Header右上角
  - 图标: 太阳(浅色) / 月亮(深色)
  - 点击切换，带平滑过渡动画

CSS实现:
  - 根元素 <html data-theme="light|dark">
  - 所有颜色通过CSS变量定义
  - 主题切换时仅改变变量值，组件自动响应
```

**设计令牌体系**：
- 间距：4px基数系统 (4/8/12/16/20/24/32/40/48)
- 圆角：2/4/6/8/12/16/24/full
- 阴影：xs/sm/md/lg/xl/2xl 六级
- 过渡：fast(150ms)/base(200ms)/slow(300ms)
- 字体：系统字体栈 + 代码字体栈

## 七、关键设计决策待讨论

| # | 决策项 | 选项A | 选项B | 建议 |
|---|--------|-------|-------|------|
| 1 | 侧边栏风格 | 始终显示（可折叠为图标模式） | 浮动覆盖式（点击打开） | **选项A** - 参考AIE固定式，更专业 |
| 2 | 面板切换方式 | 侧边栏点击切换主内容 | Tab标签页在内容区顶部 | **选项A** - 侧边栏导航更清晰 |
| 3 | 聊天界面右侧 | 固定右侧面板显示任务/工具 | 按需滑出的覆盖面板 | **选项B** - 减少视觉干扰 |
| 4 | 时间线库 | Recharts (简单) | D3.js (灵活) | **Recharts** - 更易维护，够用 |
| 5 | 关系图谱 | D3 force-directed | 独立库如react-force-graph | **D3 force** - 更灵活定制 |
| 6 | CSS方案 | 纯Tailwind类名 | CSS变量 + Tailwind辅助 | **CSS变量为主** - 更好的主题切换 |
| 7 | 图标方案 | lucide-react (已安装) | 内联SVG | **lucide-react** - 统一风格、减少维护 |
